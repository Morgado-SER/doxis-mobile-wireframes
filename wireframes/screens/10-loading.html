Purpose of this screen
  • Authentication request is in flight — credentials sent to backend, awaiting token response.
  • No back button: this is a non-interruptible state. The process must complete or timeout.
  • Tenant context chip remains visible so users know exactly which account is authenticating.
Auto-skip behaviour
  • If auth method is singular → this screen follows directly from credentials/SSO, skipping screen 07.
  • If role is singular → this screen proceeds directly to Home (screen 12), skipping screen 11.
  • In both cases, a visible loading state (this screen) prevents an abrupt jump and reassures the user.
Transitions
Auth success → Select Role auto Auth success + 1 role → Home (skip 11) auto
Design notes
  • Nav bar is intentionally absent — auth is non-interruptible once initiated.
  • Spinner uses border-top-color: var(--teal-600) against border-secondary track.
  • Error state (timeout / wrong credentials): navigate back to credentials screen with an inline error — not shown in this frame.
