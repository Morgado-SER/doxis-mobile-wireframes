Why ⋮ instead of inline buttons
  • Previous design: two 30×30px buttons with 4px gap — well below the 44px minimum touch target (Apple HIG §Layout, Material 3 §Touch targets).
  • Single ⋮ button is 44×44px — comfortable for all finger sizes, including accessibility users.
  • Scales to any number of future actions (set as default, copy credentials, share…) without changing the row layout.
  • Labeled actions in the sheet are far more readable than icon-only buttons, especially for new users.
Interactions
Tap account row body → sign in with that account tap Tap ⋮ → action sheet slides up tap Edit account → account details form tap Remove account → confirmation sheet tap Cancel / tap scrim → dismiss sheet tap Continue CTA → Signing In… tap
Action sheet rules
  • Sheet always opens with the account's name and username as a context header — user always knows which account they're acting on.
  • Edit has a supporting subtitle ("Update username, password or settings") — reduces hesitation.
  • Remove is red but also clarifies it only removes local config, not the server account — prevents panic.
  • Remove leads to a second confirmation sheet before any data is deleted — two-step destructive action.
  • Cancel button is always last, full width, ghost style — clearly the safe exit.
  • Tapping the scrim (outside the sheet) also dismisses — standard mobile behaviour.
Other states
  • Loading: tapping the row body routes to screen 10 with the chosen account context replacing "john.smith."
  • Empty state: hide list + CTA; show centred message + "Add account" primary button.
  • Delete confirmation: a second sheet — "Are you sure? This will remove John Smith from this device." + red "Remove" CTA + Cancel.
