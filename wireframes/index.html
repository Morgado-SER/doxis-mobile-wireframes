Auto-skip rules
Screen skips
  • Auth method screen skipped if tenant has exactly one method configured
  • Role screen skipped if user has exactly one role assigned
  • QR with account info skips tenant selection, goes directly to auth method
  • State B shown when all saved accounts share a single gateway URL
  • State C shown when saved accounts use multiple different gateway URLs
QR paths
  • QR with URL only → resolves to Select Tenant (screen 06)
  • QR with URL + account → skips tenant, goes to Auth Method (screen 07)
  • QR unreadable → error state (screen 03) with inline fallback
  • Cancel from QR scanner → returns to Enter URL (screen 01)
Best practice notes
  • Step indicator (1–4) on every shared-path screen so user knows where they are
  • Tenant context chip visible on auth, credentials, and loading screens
  • Auto-skip shows brief loading state rather than abrupt jump
  • Destructive CTA uses brand colour, not red — red is for errors only
